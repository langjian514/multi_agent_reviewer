▸ Agent 层 · 规则 + Qwen LLM 双轨
📊
AnalyzerAgent
规则：正则提取函数/类/依赖，圈复杂度计算
LLM：Qwen 识别语义弱点、深层架构问题
输入: 原始代码 输出: analysis_result
🔒
SecurityAgent
规则：SQL注入/XSS/命令注入/密钥泄漏/不安全依赖
LLM：Qwen 发现逻辑漏洞、权限缺陷
输入: 原始代码 输出: security_result
📏
LinterAgent
规则：行长度/命名规范/裸 except/缺少类型注解
LLM：Qwen 代码质量评级 + 改进建议
输入: 原始代码 输出: lint_result
📝
ReviewerAgent
规则：汇总所有 issues，计算质量评分 & 置信度
LLM：Qwen 生成中文综合摘要 + 优先级排序
输入: 前三 Agent 结果 输出: review_report

💬
ShortTermMemory（短期记忆）
策略：滑动窗口（默认 10 条）
功能：最近对话上下文管理
溢出一半：自动摘要后移出窗口

🗂️

存储：Milvus 向量数据库
索引：IVF_FLAT · IP 内积距离
检索：相似度 Top-K + 标量过滤
向量：Qwen Embedding → 1024 维
清理：超过 30 天自动归档
持久化 · Docker Volume
🛡️ 降级容错机制
⏱️
超时中断
每个 Agent 60s 超时
总任务 300s 上限
超时 → 走降级策略返回默认值
🔌
熔断器
连续 5 次失败 → OPEN
60s 后 → HALF_OPEN
3 次成功 → CLOSED
📋
结果兜底
每个 Agent 有预定义 fallback 数据
保证下游 Agent 永远收到有效结构
不因上游故障连锁崩溃

⚡ 关键设计决策
1️⃣
为什么用 LangGraph 而不是 Chain？
状态机天然支持条件分支（质量是否达标？）、循环（反思重试）、并行（未来可让 Independent Agent 并发执行），且每个节点状态转换清晰可追踪。
2️⃣
为什么规则 + LLM 双轨？
规则保证确定性和速度（正则 1ms 出结果），LLM 提供深度和理解力。LLM 超时或失败时自动降级到纯规则，系统不崩溃。
3️⃣
SharedState 为什么用 TypedDict？
LangGraph 传递的是字典，TypedDict 兼顾 IDE 类型提示和运行时灵活性。每个 Agent 只修改自己负责的 key，减少副作用。
4️⃣
Milvus 相比 ChromaDB 的优势？
生产级分布式向量数据库，支持标量过滤 + 向量混合搜索、IVF/HNSW 多种索引、gRPC 高性能通信，适合大规模记忆检索场景。
